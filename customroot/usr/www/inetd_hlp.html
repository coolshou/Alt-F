inetd Setup

inetd is a super-server that launches on demand other servers such as ssh or ftp.

This means that only one server (inetd) is continuously running, saving system resources such as CPU and memory for other usages; that is particularly important on underpowered computers such as the DNS-323.

The downside is that servers launched by inetd are a bit slower to start, although that is not generally human perceptible, and that the box becomes susceptible to DoS attacks. If any of these downsides is a concern, then run the servers in standalone mode.

Service: the generic service name, as it appears in /etc/services.

Program: the server program to be launched on demand.

Enable: enables the service to be under inetd control.

  • For a service to be under inetd control, the checkbox must be checked; changes are applied when the Submit button is pressed.
  • When the checkbox is grayed (disabled) this means that the service has its own configuration webUI, where it can be put under inetd control or be run as a standalone server (in this case it will appear under Services->Network)

Configure: allows to specifically configure the server program.
