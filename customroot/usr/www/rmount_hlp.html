Remote NFS and CIFS Imports Setup Help

In this page you can specify remote NFS and CIFS shares to become available to the box.
A share is a network offer of an existing folder.

  • Shares to import from other NFS servers

    Specify remote folders to be locally mounted, e.g. to make them available under other protocol.
    • Disable: Check to temporary stop mounting the share..
    • Host: The remote hostname (or IP address) that exports the share.
    • Folder: The remote folder to mount locally.
    • Discover: Hit to NFS browse the network; hitting a line will fill the Host and Folder fields.
    • Local Folder: Local folder where the remote share should be made available. It is recommended to use an empty folder directly under "/mnt"; if necessary create the folder, e.g. "/mnt/remoteNFS".
    • Search: Hit to browse the local folder structure (and possibly create the mount folder).
    • Mount Options: NFS mount options, comma separated, no spaces.
    • Options: Opens a new window with a list of the most frequently used mount options and the currently selected ones; hit CTRL while clicking to select multiple options. Options of the form option=value does not preserve the current value.

    • Delay NFS start on boot:

      NFS stores some client information on disk, for recovery on reboot in case of server or client crash. That information is stored in a sub-directory where Alt-F packages are stored. At boot time, when NFS starts, the Alt-F directory might not yet be available and such recovery information is missing.
      Checking this checkbox delays the NFS start until the Alt-F directory becomes available. Notice that if the checkbox is checked and the Alt-F directory never appears, then NFS will not auto-start.

  • Shares to import from other CIFS servers

    You can make other computer shares available to the box.
    • Host - the computer name where the share is available.
      If empty, the whole entry will be removed when the Submit button is hit.
    • Share - the remote share name.
      If empty, the whole entry will be removed when the Submit button is hit.
    • Discover - opens a new window and performs a SMB/CIFS network scan, showing the existing computers and offered shares; clicking a share will fill-in the previous Host and Share entry fields.
    • Local Folder - the local folder where the remote share contents will be available.
      If the folder is not empty, its contents will (temporarily) disappear while the remote share is mounted, and will reappear when the remote share is unmounted.
    • Search - opens the Folders Browse window, which will fill-in the Local Folder entry field with the selected folder.
    • Mount Options - specify the desired mount options; the default values will generally be OK.
After an entry is filled-in and the Submit button is hit, a Mount button will appear, which will make the remote share locally available; when mounted, an unMount button will appear instead, which if selected will unmount (unmap) the remote share.
